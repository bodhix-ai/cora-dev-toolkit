## 1. User Personas

### 1.1 Regular User (Workspace Member)

**Description:** Team member who uses AI chat for work tasks within workspace context.

**Goals:**
- Get AI-assisted answers grounded in organizational knowledge
- Reference documents without manual searching
- Share useful conversations with teammates
- Access conversation history quickly

**Pain Points:**
- Generic AI responses without organizational context
- Difficulty finding relevant documents
- Lost conversation context between sessions

### 1.2 Personal User

**Description:** Individual user who wants personal AI conversations not tied to workspaces.

**Goals:**
- Quick exploratory conversations
- Personal notes and research
- Privacy for sensitive questions

**Pain Points:**
- Mixing personal and work conversations
- Cluttered conversation list

### 1.3 Power User

**Description:** User who heavily uses AI chat with multiple KB sources and advanced features.

**Goals:**
- Fine-grained control over KB grounding
- Organize conversations efficiently
- Track token usage

**Pain Points:**
- Too many clicks to configure KB sources
- Difficulty managing many conversations
